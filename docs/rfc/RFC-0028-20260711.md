# RFC-0028: Migrate from REST to GraphQL for Client-Facing APIs

## Summary

This RFC proposes migrating our client-facing REST APIs to GraphQL to reduce over-fetching, eliminate N+1 query problems on the client side, and provide a self-documenting API schema.

## Motivation

Mobile clients over-fetch data (receiving 40+ fields when needing 5). Multiple round trips needed to assemble complex views. No built-in way to express data requirements.

## Detailed Design

Phase 1: Schema Design (Week 1-2). Phase 2: Resolver Implementation with DataLoader (Week 3-4). Phase 3: Parallel running alongside REST (Week 5-8). Phase 4: REST deprecation with 6-month notice (Week 9-12).

## Alternatives Considered

REST with sparse fieldsets: partial solution. gRPC: better performance but poor browser support. tRPC: excellent DX but couples client and server.
